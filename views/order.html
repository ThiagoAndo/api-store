Order Route

Order route features and usage:

  • To request a new order the client must have at least one product on their cart.
  • User id must be provided.
  • To request a new order as a guest, a cart must be provided with a product. User id does not need to be provided.
  • Name must be passed as first name and last name only.
  • Email must be a valid email address format.
  • The user will get an email with the purchase details.

GET /user order

Returns all orders from a specific user.

Example Request:

GET https://next-store-api-62m2.onrender.com/order/{USER ID}

Example Response:

POST /new order

Registers a new user's order.

Example Request:

POST https://next-store-api-62m2.onrender.com/order/

Request Body:

{
        
 "id":"4o40o1vlyltmkxk",
 "name":"Stephen King",
 "email":"stephenking@lorem.com"
        
}

Example Response 201: 

{ message: "Invoice created" }
Example Response 407: 

{ message: "Incomplete Body" }
{ message: "An error has occurred"}
{ message: "Name is not valid. Make sure to enter first and last name only"}
{ message: "Email is not valid"}
