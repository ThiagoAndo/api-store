Address Routes

Address route features and usage:

  • User id must be provided.
  • A complete JSON contaning all fields and values must be passed to the API.

GET /address

Returns user address.

Example Request:

GET https://next-store-api-62m2.onrender.com/add/{USER ID}

Example Response 200:

[
        {
        "id": "4jgm2z11gfjlyrfbmsl",
        "line_one": "356, ashgrove",
        "line_two": "castleknock",
        "town_city": "dublin 24",
        "constry_state": "Dublin"
        }
]

Example Response 404: 

{ message: "There is no user with id: XXXX"}



POST /address

Register an address of a user.

Example Request:

POST https://next-store-api-62m2.onrender.com/add

Request Body:

{
        "id": "4jgm2z11gfjlyrfbmsl",
        "line_one":"356, ashgrove",
        "line_two":"castleknock",
        "town_city":"dublin 24",
        "constry_state":"Dublin"
        }

Example Response 201:

{message: "Address created successufuly"}

Example Response 404: 
{ message: "Incomplete Body" }
{ message: "There is no user with id: XXXX"}



PATCH /user address

Update user address.

Example Request:

PATCH https://next-store-api-62m2.onrender.com/add

Request Body:

{
        "id": "4o40o1vlyltmkxk",
        "line_one": "221 baiepau",
        "line_two": "Vl. norma",
        "town_city": "São Paulo",
        "constry_state": "SP"
}

Example Response 201:

 { message: "Updated user detail with id XXXX"}

Example Response 404: 
{ message: "Incomplete Body" }
{ message: "There is no user with id XXXX"}
